Summary of results

Model inputs

Three main modelling methods were attempted (generalised linear modelling (GLM), using a quasibinomial model with a logit link function; random forest (RF) model modelling the outcome; and a random forest model that models the change in the outcome from the previous year).

Variations on model inputs were also tested. Models were tested where the input data varied between 3 and 7 years. For each of those, different amount of lagging to the input data was tested too - in order to see whether there was a knock on effect of a previous year for an input variable on the outcome variable for the following year. Models with 0, 1 and 2 lagged years were tested. The importance of the previous yearâ€™s outcome value as a predictor variable was also tested, by either including it or not including it in the modelling process. See Table 1 for a summary of the combinations of inputs.

In all, nearly 450 different models were built and assessed. In reality, some of the time series for the outcome variables were not long enough to test for all variations of three to seven years, so fewer than 450 models were built.

Model selection

Each of the outcome variables are a proportion, so they have a scale of between 0 and 1. Despite the consistent scale, the distribution between within the scales of the outcome variables differs. MAPE was used (as described in Model evalaution) as a comparable metric across the different models.

Baseline comparison

As described in Model evalaution, two baseline naive methods of estimating future performance were performed and compared with the more sophisticated methods to contextualise the performance. The two methods chosen to compare with were selected because of their simplicity, but also because these are typical methods teams might employ for predicting future performance:

  • Next yearâ€™s performance is equal to the previous yearâ€™s performance
  • Next yearâ€™s performance is equal to a linear extrapolation based on the past 3 yearsâ€™ performance
